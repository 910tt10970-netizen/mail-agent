# email_writer — агент «Автор e-mail рассылки»

## Краткое описание

Агент принимает URL статьи, извлекает текст и создаёт email-рассылку через **OpenAI-совместимый** Chat Completions API (официальный OpenAI или прокси: LiteLLM, OpenRouter, Azure-шлюз и т.д., если они отдают тот же формат эндпоинта `/v1/chat/completions`).

## Что генерирует агент

- тему письма;
- preheader;
- тело письма;
- CTA с предложением позвонить менеджеру компании для организации встречи.

## Важные требования к результату

- письмо всегда на русском языке;
- стиль экспертный и деловой;
- тема до 80 символов;
- preheader до 120 символов;
- тело письма от 900 до 1500 символов;
- результат возвращается обычной строкой с блоками.

## Структура проекта

В проекте используются строго четыре файла:

```
email_writer/
├── agent.py
├── openai_module.py
├── requirements.txt
└── README.md
```

Дополнительные файлы и папки в первом варианте проекта не используются: нет тестов, примеров `.env`, пакетных `__init__.py` и прочих элементов вне этого набора.

## Установка зависимостей

Из каталога, где лежит папка `email_writer` (родительский каталог относительно `requirements.txt`):

```bash
pip install -r email_writer/requirements.txt
```

## Настройка API-ключа

Через переменную окружения (пример для Unix-подобных оболочек):

```bash
export OPENAI_API_KEY="your_api_key_here"
```

Дополнительно можно указать **`API_KEY`** (если удобнее единое имя в `.env`) — используется, только если `OPENAI_API_KEY` не задан.

### Прокси и другая модель (OpenAI-совместимый API)

Если запросы идут через прокси или другой хостинг моделей с совместимым REST API, задайте базовый URL (часто заканчивается на `/v1`):

```bash
export OPENAI_BASE_URL="https://your-proxy.example/v1"
# необязательный альтернативный ключ переменной:
# export OPENAI_API_BASE="https://your-proxy.example/v1"

export OPENAI_MODEL="provider/model-name"
# или то же самое:
# export LLM_MODEL="provider/model-name"
```

Если `OPENAI_BASE_URL` не указан, клиент ходит на сервер OpenAI по умолчанию. Если не указаны `OPENAI_MODEL` / `LLM_MODEL`, используется модель по умолчанию из кода (`gpt-4o-mini`).

Пакет `python-dotenv` подключён в коде: при желании можно создать файл `.env` рядом с проектом и указать в нём переменные. Файл `.env` в репозиторий не входит и не является частью заявленной структуры из четырёх файлов — пользователь создаёт его сам.

## Пример использования в Python

Убедитесь, что каталог, содержащий пакет `email_writer`, доступен в `PYTHONPATH` (например, запуск из корня проекта, где лежит папка `email_writer`).

```python
from email_writer.agent import run

email_text = run("https://example.com/article")
print(email_text)
```

## Ожидаемый формат результата

Ответ — одна строка (или текст с переносами строк) примерно в таком виде:

```
Тема: ...

Preheader: ...

Тело:
Здравствуйте!

...

CTA: Позвоните менеджеру нашей компании, чтобы организовать встречу и обсудить, как применить эти идеи в вашей ситуации.
```

## Возможные ошибки

- невалидный URL;
- недоступная страница;
- HTTP-ошибка при загрузке;
- слишком короткий текст статьи (после извлечения и очистки);
- отсутствует ключ (`OPENAI_API_KEY` или `API_KEY`);
- ошибка OpenAI API;
- таймаут или лимит запросов (в т.ч. при повторных попытках в модуле OpenAI).

## Основная функция

**`run(url: str) -> str`** — валидирует URL, загружает страницу, извлекает текст статьи, очищает его, отправляет в Chat Completions (OpenAI или совместимый прокси по `OPENAI_BASE_URL`) и возвращает готовую email-рассылку строкой.
